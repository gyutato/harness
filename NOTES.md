# 하네스 재사용 추출 로그

여러 하네스에서 같은 방법론이 반복되는지 관찰하고, 추출 시점을 판단하기 위한 로그.

## 추출 기준

- 2+ 하네스에서 **같은 방법론이 실제 구현된 뒤** 추출 판단
- 3번째에서도 같은 모양이면 강한 신호 (Rule of Three)
- 추출 그릇은 **skill**이 기본 — Claude Code의 skill 정의("절차적 지식/방법론 재사용")와 일치
- Boilerplate 공유(정체성 선언, 팀 통신 등)는 추출 대상 아님 — 이건 Claude Code 메커니즘의 노출이지 하네스가 만드는 단위가 아님

## 현재 관찰된 후보 (리서치 하네스 1개 — 추출 근거 없음, 기록만)

| 후보 | 출처 에이전트 | 재등장 예상 시나리오 |
|------|-------------|------------------|
| 외부 자료 교차 수집 (웹·커뮤니티·코드·다국어) | context-scout | 조사형 하네스 일반 |
| 후보 탐색 + 근거 기반 추림 | paper-hunter | 대상 선별이 필요한 조사 하네스 |
| 여러 입력 비판적 통합 + 구조화 보고서 | insight-analyst | 리서치/리뷰/감사 하네스 |
| 리더의 사용자 중계 + 권한 차단 루프 | research-lead | 팀 모드 사용하는 모든 하네스 |
| 구조화된 심층 독해 | deep-reader | 장문 자료(논문/RFC/문서/제안서) 분석 하네스 |

## 체크 시점

- **새 하네스 시작 시**: 위 후보와 겹치는 방법론이 필요한지 훑어보기
- **새 하네스 완성 후**: 실제로 위 후보와 같은 일을 한 섹션이 있었는지 식별 → 있으면 "사례 2"로 이 표 갱신
- **사례 2+ 쌓이면**: 실제 추출 판단. 세 번째까지 누적되면 강한 추출 신호
